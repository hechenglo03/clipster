核心模块设计

六个关键模块的职责与实现要点

⧉

ClipboardWatcher

轮询 NSPasteboard.changeCount，0.5s 间隔检测变化。读取文本/URL/图片/文件多类型，发送到处理管线。后台静默运行，无 UI 干扰。

⬚

ContentClassifier

规则 + 正则识别内容类型：URL、邮箱、颜色值（#hex）、代码块（缩进/关键字）、文件路径。输出 Category 枚举，驱动分类视图。

⚡

SearchService

基于 SQLite FTS5 全文索引，支持模糊匹配、按分类过滤、按时间排序。键入即时查询（debounce 80ms），万条记录 <10ms。

⌘

HotkeyManager

CGEventTap 注册全局快捷键，监听 keyDown 事件。支持自定义绑定、冲突检测（比对系统快捷键表）、运行时动态重注册。

↻

PasteSimulator

写入 NSPasteboard 后，通过 CGEvent 模拟 ⌘V 按键事件，把内容粘贴到当前焦点应用。处理权限申请与沙盒限制。

⛁

ItemRepository

统一数据访问入口，封装 GRDB 操作。提供增删查改、分页加载、收藏置顶、批量清理、过期自动清理（默认保留 30 天 / 1000 条）。

数据模型

SQLite 表结构 + FTS5 全文索引

schema.sql · 主表
CREATE TABLE clip_item (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  content     TEXT NOT NULL,        -- 文本内容
  content_hashTEXT UNIQUE,         -- SHA256 去重
  category    TEXT NOT NULL,        -- text/link/...
  mime_type   TEXT,                    -- 内容 MIME
  payload_url TEXT,                    -- 图片/文件路径
  size_bytes  INTEGER,
  is_favorite INTEGER DEFAULT 0,
  is_locked   INTEGER DEFAULT 0,     -- 私密加密
  app_source  TEXT,                    -- 来源 App bundleID
  created_at  REAL NOT NULL,         -- 时间戳
  pinned_at   REAL
);
CREATE INDEX idx_cat ON clip_item(category);
CREATE INDEX idx_time ON clip_item(created_at DESC);
fts.sql · 全文索引
-- FTS5 虚拟表，支持中文分词
CREATE VIRTUAL TABLE clip_fts
USING fts5(
  content,
  category,
  tokenize = 'simple'
);

-- 触发器：自动同步索引
CREATE TRIGGER ai_item AFTER INSERT ON clip_item BEGIN
  INSERT INTO clip_fts(rowid,content,category)
  VALUES (new.id,new.content,new.category);
END;

-- 查询：模糊搜索 + 分类过滤
SELECT c.* FROM clip_fts f
JOIN clip_item c ON c.id = f.rowid
WHERE clip_fts MATCH :kw
  AND c.category = :cat
ORDER BY c.created_at DESC LIMIT 50;

关键技术实现

三个最难啃的硬骨头及对应方案

①

全局快捷键

macOS 无原生全局热键 API，需用 CGEventTap 监听系统事件流。

HotkeyManager.swift
let mask = CGEventMask(
  (1 << CGEventType.keyDown.rawValue)
)
let tap = CGEvent.tapCreate(
  tap: .cgSessionEventTap,
  place: .headInsertEventTap,
  options: .defaultTap,
  eventsOfInterest: mask,
  callback: { _, _, event, _ in
    // 比对 modifierFlags + keycode
    if HotkeyManager.match(event) {
      event = nil // 吞掉
      Window.show()
    }
    return Unmanaged.passRetained(event)
  }, userInfo: nil)
RunLoop.add(tap, .common)

⚠ 需「辅助功能」权限（Accessibility）

②

剪贴板监听

NSPasteboard 无回调，需轮询 changeCount 检测变化，0.5s 间隔兼顾性能与及时性。

ClipboardWatcher.swift
class ClipboardWatcher {
  private var lastCount: Int = 0
  private let pb = NSPasteboard.general

  func start() {
    Timer.scheduledTimer(
      withTimeInterval: 0.5,
      repeats: true
    ) { _ in
      let cur = self.pb.changeCount
      guard cur != self.lastCount else { return }
      self.lastCount = cur
      let item = self.read()
      Pipeline.process(item)
    }
  }
}
③

自动粘贴到目标 App

写入剪贴板后，需模拟 ⌘V 把内容粘到当前焦点应用，关键是要在窗口隐藏后还原焦点。

PasteSimulator.swift
func paste(_ item: ClipItem) {
  // 1. 写入剪贴板
  pb.clearContents()
  pb.setString(item.content, forType: .string)

  // 2. 隐藏面板，还原前一个 App
  NSApp.hide(nil)
  NSApp.yieldActivation(
    toApplicationWithOptions: .default)

  // 3. 延迟 60ms 模拟 ⌘V
  DispatchQueue.main.asyncAfter(
    deadline: .now() + 0.06) {
    CGEvent.postCmdV()
  }
}

核心数据流

内存优化

图片存文件系统，DB 只存路径；列表虚拟化（NSCollectionView 重用 Cell）；常驻仅 30MB。

索引优化

FTS5 全文索引 + created_at DESC 复合索引；分页加载 50 条/页；查询 debounce 80ms。

自动清理

后台定时任务，默认保留 1000 条 / 30 天；超额自动淘汰最旧；可配置保留策略。
