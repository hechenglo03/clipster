固定列表 · Pin Board

把最常用的内容钉在指尖，一键直达，永不沉入历史

功能定位

“收藏”是按分类聚合，“固定”是按使用频率置顶。固定内容不参与自动清理，永远排在最前面。

📌

一键固定 / 取消

选中条目后按 P 或右键「固定到顶部」，立即置顶；再次按 P 取消固定，回到正常时间序。

⌨️

数字键优先

固定条目在列表中始终占据 1-9 号数字键。未固定条目从 10 开始编号，高频内容盲打即粘贴。

🛡

不被清理

固定条目默认不参与 30 天自动清理，也不计入保留 1000 条的限制，重要信息长期保存。

⬚

独立 Pin 面板

侧边栏新增「固定」分类，只显示固定条目；也可在主列表顶部把 Pin 区展开 / 折叠。

☁

拖拽排序

在固定面板内拖拽调整顺序，自定义你最顺手的快捷键位置。

⛁

持久化

固定状态写入 SQLite，按 pinned_at 排序，重启应用顺序不变。

数据模型

最小改动：复用现有 clip_item 表，新增 pinned_order 字段

ALTER TABLE clip_item ADD COLUMN pinned_order INTEGER DEFAULT NULL;
ALTER TABLE clip_item ADD COLUMN is_pinned INTEGER DEFAULT 0;

-- 查询固定列表
SELECT * FROM clip_item
WHERE is_pinned = 1
ORDER BY pinned_order ASC, pinned_at DESC;

-- 查询主列表（固定置顶 + 普通时间序）
SELECT * FROM clip_item
WHERE category = ?
ORDER BY is_pinned DESC, pinned_order ASC, created_at DESC;
is_pinned 布尔 pinned_order 排序 pinned_at 时间戳 不参与自动清理

实现流程

四步把固定列表接入现有架构

1

数据层

新增 is_pinned / pinned_order 字段，Repository 增加 pin/unpin/reorder 方法。

2

UI 层

侧边栏新增「固定」分类；列表顶部增加 Pin 分隔区。

3

快捷键

监听 P 键，PanelController 转发到 ItemRepository。

4

清理策略

自动清理任务排除 is_pinned = 1 的记录。

与现有“收藏”的区别
